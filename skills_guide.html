これは何か

skill は、Claude Code に「決まった手順・方針」を覚えさせておき、/名前 で呼び出せる再利用可能なワークフロー。毎回同じ指示を書かなくても、狙った動きをさせられる。

ここに載っているのは、あなた(Shin)の個人開発・X発信・日々の運用に合わせて作った 自作skill 5本。プロジェクトを横断して使えるよう、すべてユーザー共通の場所に置いてある。

agentとの違い: agent(サブエージェント)は「別働隊に丸ごと任せる」仕組み。今回作ったのは全部 skill で、agent はまだ0本(候補は後述)。

置き場所とスコープ(どこで使える?)

使える範囲は置き場所で決まる。今回の5本はすべて一番上の「グローバル」に置いてある = このMacの全ディレクトリで有効。

置き場所使える範囲今回
~/.claude/skills/<名前>/SKILL.md
グローバル / ユーザー共通		全ディレクトリ・全プロジェクト。どこで開いても /名前 が出るこれ
<プロジェクト>/.claude/skills/…
プロジェクト固有		そのプロジェクト配下でのみ表示—
プラグイン経由プラグインを入れた環境—
優先順位: 同名 skill がグローバルとプロジェクト両方にあると、プロジェクト側が優先。特定プロジェクトだけ挙動を変えたい時は、そのプロジェクトの .claude/skills/ に同名で置けば上書きできる。

使い方(共通ルール)

呼び出し

チャットで /コマンド名 と打つ。引数を付けられるものは /x-draft ScreenTime APIの話 のように続ける。

反映のタイミング

新しく作った/編集した skill は セッション開始時に読み込まれる。今サジェストに出ないときは Claude Code を一度立ち上げ直すと、次回から全プロジェクトで有効になる。

編集

各 skill の中身は SKILL.md(Markdown)。方針を変えたくなったら該当ファイルを直接編集すればいい。追加・編集の仕方を参照。

✍️ /x-draft — X投稿草案

/x-draftX投稿ドラフト生成

X投稿の草案を、淡々・ドライなビルダー調で3〜5案生成する。ポエミー禁止・技術のハマりどころを淡々と出す文体ルールを内蔵。

いつ使う
ツイート作って今日のポストこれ発信したいスレにして
入力
トピックを引数で渡す。引数なしなら、今いるプロジェクトの直近git活動(コミット・差分)から自動でネタを拾う。
出力
単発3本(報告 / 技術ハマりどころ / 小ネタ)＋スレ案1本。各案に推定文字数付き。
保存先
採用案は make_idea/sns_marketing/posts_log.md に追記(任意)。
ファイル
~/.claude/skills/x-draft/SKILL.md
>/x-draft
>/x-draft Re:Nightのロック維持をScreenTime APIで実装した話

🌅 /morning — 朝ブリーフィング

/morning1日の頭出し

今日の予定・要対応メール・「今日やるべき一手」を1画面にまとめる。専業移行に向けた日次運用用。

いつ使う
朝ブリーフィング今日の予定今日やることmorning
集める情報
① Googleカレンダーの今日の予定 ② 要対応の未読Gmail(広告・通知は除外)3〜5件 ③ 総合ロードマップ.html 由来の「今日の一手」1〜3個。
依存
Googleカレンダー / Gmail の MCP 連携(初回は認証)。取れない情報は「取得できず」と明記してスキップ。
ファイル
~/.claude/skills/morning/SKILL.md
>/morning
毎朝自動化したい場合: 手動実行が一番確実(ローカルのMCP認証＋ローカルファイルを両方使うため)。定期化するなら /schedule(クラウドcron)や /loop を使う。ただしクラウド実行ではカレンダー/Gmailの認証やローカルファイルが届かないことがあり、その場合はロードマップ中心の縮小版になる。運用が固まったら設定を依頼。

🧭 /tackle — 難タスクの作業規律

/tackle分解→実行→検証→次決め

難しい・曖昧・大きい多段タスクに規律をもって取り組む。勢いで一気にやらず、検証可能な小さい前進の連鎖に変える。Opus 4.8向け。

いつ使う
これ難しいから整理して進めてどう分解する?大きめのタスク設計から実装まで
流れ
0.見極め(ゴール1文・完了の証拠・不可逆操作の確認)→ 1.分解(各段に完了条件を1文で・未知を先に潰す)→ 2.実行(一度に一段)→ 3.検証(3レベル: 機械的/反証/独立レビュー・証拠つきで報告)→ 4.次決め(状態→行動の表で判断・同じ所で2連敗したら手を変える)→ 5.報告(やったこと/検証/残りを正直に)。
Opus 4.8
並列サブエージェントで枝を同時進行、大規模コンテキストで先に広く把握、超大規模はWorkflow検討。
使わない時
1〜2手の単純作業や会話のみ(オーバーヘッドの方が大きい)。
ファイル
~/.claude/skills/tackle/SKILL.md
>/tackle Re:Nightの課金導線を設計から実装まで通したい

🔁 /tackle-retro — tackleの自己監査・改善ループ

/tackle-retroskillを育てる仕組み

/tackle を使った作業の直後に自己監査し、破られた規律を記録 → 溜まったら SKILL.md をパッチする改善ループの実行部。skillは書いて終わりではなく、破られた記録で育てる。

いつ使う
ふりかえりレトロtackleの監査skillを改善したい
動作
①各段(0〜5)を証拠つきで監査(守れた/破った/該当なし)②tackle/LOG.md に1行記録 ③newが5件以上 or 同一原因2回でパッチ提案(足すだけでなく削る)→ 承認後に適用＋gitコミット。
記録先
~/.claude/skills/tackle/LOG.md(日付/タスク/破った規律/原因/パッチ候補/状態)。
ファイル
~/.claude/skills/tackle-retro/SKILL.md
>/tackle-retro
運用: /tackle で作業 → 終わったら /tackle-retro → ログが溜まったらパッチ提案が自動で出る。skillの版管理は ~/.claude/skills の git が担う(改悪したら戻せる)。

🧠 /fable — Fable 5 流の調査と推論

/fable認知様式(Opus 4.8でFable 5風)

最新世代モデルが既定で行う「調べ方 → 結論の出し方 → 伝え方」を指示追従で再現する。中心原則は1つ: 証拠 → 確信度 → 結論。逆走(結論が先)は禁止。監査→3視点草案→3審査員→統合のworkflowで作成。

いつ使う
深く考えてFable風にじっくり調べてちゃんと裏取りして本当に合ってる?
流れ
仮説(反証可能な仮説1〜3個を先に宣言)→ 観測(委譲・並列サーチ・部分読み・一次情報・記憶より現物)→ 打ち切り(揃ったら即行動・同じ切り口2回空振りで変える)→ 結論(複数解釈の保持・確度ラベル [事実]/[推論]/[推測]・反証を先に)→ 伝達(結論ファースト・誇張なし・推奨は1つ)。
/tackleとの関係
/tackle が「次にどの段をやるか」(工程管理)、/fable が「その段の中でどう考え・調べ・結論するか」(認知様式)。併用可。軽い応答でも「確度ラベル+誇張なし」の2点だけは常時有効。
ファイル
~/.claude/skills/fable/SKILL.md
>/fable このバグの原因をちゃんと裏取りして特定して

🔧 追加・編集の仕方

新しい skill を足す

フォルダを1つ作り、その中に SKILL.md を置くだけ。フォルダ名がコマンド名になる。

$~/.claude/skills/<名前>/SKILL.md

SKILL.md の先頭はこの形(description に「いつ使うか」を書くほど呼び出しの精度が上がる):

フィールド内容
nameコマンド名(フォルダ名と一致)
description何をする skill か＋どんな依頼の時に使うかのトリガー語

既存を直す

該当の SKILL.md を編集するだけ。編集後はセッションを立ち上げ直すと反映。

版管理・他PCでの利用

~/.claude/skills は gitリポジトリ(2026-07-07〜)。リモートは非公開の github.com/hshinya/claude-skills。変更したらコミット+pushしておくと、改悪した時に戻せる&他PCと同期できる。

他のPCで /tackle・/tackle-retro だけ使う場合(Claude Codeログイン済み前提):

$git clone --sparse https://github.com/hshinya/claude-skills.git ~/.claude/skills
$git -C ~/.claude/skills sparse-checkout set tackle tackle-retro

全部使うなら普通に git clone するだけ(フォールバック追加済みなのでどのPCでも壊れない)。以後の同期は このMacで push → 他PCで git pull。

「〇〇な skill 作って / この skill こう直して」と言えば、この様式に沿って作成・編集します。
