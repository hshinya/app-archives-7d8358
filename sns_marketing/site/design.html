DESIGN GUIDE

デザイン・画像作成ガイド

デザイン未経験エンジニアのための、最小工数で「伝わるビジュアル」を作る実践ガイド。ツール選定・サイズ早見表・用途別ワークフローをこの1ページに集約。

詳細・出典は research/07_design-and-images.md を参照。サイズ等の数値はApple公式ページ等の一次情報で確認済み(2026年6月時点)。

0. 結論(まずこれだけ)

✅ デザイン未経験でも、この3点だけ押さえればOK
  • 最初の3ヶ月はCanva無料プラン一本で十分。FigmaはアプリUIモックを作る段階で追加
  • SNSの主力ビジュアルは「作り込んだ画像」ではなく「生スクショ+デモGIF」。エディタ・ターミナル・Before/Afterのスクショはデザインスキルゼロで今日から出せて、テキストのみの約3倍の反応を取れる
  • デザイン学習は『ノンデザイナーズ・デザインブック』の4原則(近接・整列・反復・対比)の章だけ読む。あとはテンプレを崩さず使う

用途ツール費用

本格デモ動画(ズーム・カーソル強調) Screen Studio(年払い月$9、Mac専用、iPhone実機USB録画対応) 反応が出てから

⚠️ Canvaの注意

素材・テンプレの無加工販売、素材単体の再配布、Canva素材を含むロゴの商標登録はNG。

⚠️ AI画像生成の使い分け(2026年時点)
  • アイコン叩き台 = Recraft(SVG出力可)
  • 文字入り = Ideogram v3(ただし日本語は不安定 → 文字はCanvaで載せる)
  • 法的安全最重視 = Adobe Firefly(商用利用補償付き)

無料プランは「商用不可」「生成物全公開」など条件差が大きいので使用直前に必ず規約確認。

3. 用途別ワークフロー

そのままスクショ

エディタ/ターミナル/アプリ画面をそのままスクショ。

動きがあるものは画面録画→GIF化

10〜60秒、ネイティブアップロード。

加工はしない

Build in Public文化では生スクショが正解。

テキストで構成を書く

1図解=1メッセージ、要素は5〜7個以内。

Napkin AIで叩き台生成

テキストから図解を自動生成して土台にする。

Canvaで仕上げ

配色2〜3色・フォント1〜2種の自分ルールを守る。

スマホで読める文字サイズか確認

投稿前に必ずモバイル表示でチェック。

図解は保存・リポストされやすい(ブックマーク率を記録して効果測定)。

型を守る

3秒で課題 → 10〜20秒デモ → CTA。30秒以内・字幕付き・縦型。

冒頭3秒で機能を見せる

Tony Dinh式: 新機能ごとに短いスクリーン録画を投稿。

同じ素材を横展開

X → リール → Shorts → TikTok に転用。ウォーターマーク付き動画は他プラットフォームで降格されるため必ず元素材から書き出す。

テンプレに画面ハメ+コピー載せ

AppMockUp Studio / Screenshots Pro / AppLaunchpad(無料)を使う。

スクショ1〜3枚目で価値を伝え切る

検索結果に表示されるのは1〜3枚目。A/Bテストで中央値+11.8%のCVR改善。

アイコンはAI生成→Canvaで調整

叩き台はRecraft/Ideogramで生成 → Canvaで調整(商標登録予定があるならAI生成・Canva素材のみは避ける)。

Xの反応が良かった投稿を翻訳

8〜10枚構成(フック → 展開 → まとめ → CTA)に組み直す。

テンプレ or 自動変換

Canvaテンプレ or Contentdrips/PostNitroで自動変換(1セッション10〜20分)。

保存・DMシェアを誘発する設計

1枚目フック、最終枚CTA、ハッシュタグ3〜5個。

4. 学習リソース(最小限)

リソース読む範囲
『ノンデザイナーズ・デザインブック』 4原則(近接・整列・反復・対比)の章のみ
『なるほどデザイン』 余裕があれば
catnose氏のリリース告知ポスト スクショ+デモGIFの「最小で美しい」見本として観察
💡 運用ルール

配色は2〜3色、フォントはNoto Sans JP等1〜2種に固定。テンプレを「選ぶ」のはセンス不要、テンプレを「いじる」のがセンス必要 — だからいじらない。
