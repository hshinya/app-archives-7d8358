3行サマリー

1

Xは「導線」である

Xは「フォロワーを増やす場」ではなく「見込み顧客100人に届けてウェイトリストに登録してもらう導線」。アルゴリズムは会話(リプライ)最重視で、最初の1,000人まではリプライ7割:投稿3割。

2

「反応」と「需要」は別物

「SNSの反応」と「需要」は別物。いいねではなく、ブックマーク・DM・ウェイトリスト登録・前金だけを信じ、合格ラインを事前に決めてから検証する(不合格なら作らない=無駄な実装の回避)。

3

仕組みで続ける

効果実感まで4〜6ヶ月。気合いではなく仕組み(1日45分上限・日曜バッチ・最低ライン週3ポスト)で続ける。大半が2ヶ月で挫折するので、継続自体が参入障壁。

戦略の核(これだけは外さない5原則)

ロードマップ全体は、調査から導かれた次の5原則の上に組み立てている。

1

Xはアプリへの「導線」であり、フォロワー数はKPIではない。

フォロワー数と売上はほぼ無相関(相関0.04の調査あり)。ぶべ氏はXフォロワー約4,000人でMRR30万円を達成している。目標は「1万フォロワー」ではなく「見込み顧客100人に届き、ウェイトリストに登録してもらうこと」。

2

2026年のXアルゴリズムは「会話」が全て。

エンゲージメントの重みは いいね0.5 < リポスト1 < ブックマーク10 < リプライ13.5 < 著者との会話往復75(2023年公開コード由来の序列だが方向性は現在も有効)。フォロワー1,000人までは「自分の投稿を磨く」より「他者への質の高いリプライ」が成長の主エンジン(70/30ルール)。外部リンクは本文に貼らない(リーチがほぼゼロになる)。

3

「SNSの反応」と「需要」は別物。2つのダッシュボードで測る。

プログラミング×AI×個人開発で集まるフォロワーは同業者であり、一般消費者向けアプリの顧客ではない。Xは「認知・応援・フィードバック」用。需要(支払い意思)の検証は、シグナル強度の序列(いいね < 投票 < ブックマーク < 具体的リプライ < DM < ウェイトリスト登録 < 前金)の右側と、LP+少額広告・Mom Test式インタビューで行う。

4

今はBuild in Publicの「フルオープンが最適」なフェーズ。

アイデア盗用が起きるのは$10K MRR(月150万円)以降で、収益ゼロの段階で盗まれるものはない。$10K MRRまでは数字も過程も全公開→以降は非公開化、という段階戦略をあらかじめ決めておく。顔出し・実名は不要(catnose式のハンドルネーム+固定アイコンで可)。

5

気合いではなく仕組みで続ける。開発:SNS = 2:1以上を死守。

SNSは1日45分上限。「開発でやったことを投稿する」(開発ファースト原則)。効果実感まで4〜6ヶ月かかるのが正常であり、最初の90日は反応ゼロが仕様。大半の人は2ヶ月で挫折するため、継続自体が参入障壁になる。

全体マップ(Phase 0〜5)

フェーズ 期間目安 テーマ 主KPI 出口条件
Phase 0 今週〜2週間 アカウント・環境・企画の準備 準備チェックリスト完了 初投稿+固定ポスト公開
Phase 1 1〜2ヶ月目 発信の習慣化と土台づくり 行動KPIのみ(投稿数・リプ数・継続率) 45分ルーチンが3週間途切れず回る
Phase 2 2〜4ヶ月目 アイデア検証と投稿の質の確立 ER2%以上、検証シートの完了数 作るアプリが1本決まる(or 撤退判断)
Phase 3 4〜6ヶ月目 実装+プレローンチ(並行) ウェイトリスト登録数(まず25→100人) ストア審査提出+事前登録開始
Phase 4 リリース週〜1ヶ月 ローンチと初速 初週300DL・月1万円 リリース後記事公開+改善サイクル稼働
Phase 5 6ヶ月目以降 グロースと複利 MRR、2本目の検証開始 —
ℹ️ 期間の前提

期間は「兼業・1日合計2〜3時間(開発含む)」前提の目安。Phase 2と3は検証結果次第で前後し、検証に失敗したらPhase 2を繰り返す(それが「無駄な実装をしない」ことの実体)。
