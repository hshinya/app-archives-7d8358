X運用の半自動化(Skills / Agents / Hooks)

戦略を「毎日回る仕組み」にするための自動化セット。情報収集と情報発信の下書きをClaude Codeが担当し、投稿・リプライは人間が手動で行う(human-in-the-loop)。X APIは使わない=コスト0円・凍結リスク0。

設計思想(なぜこの形か)

1

投稿は必ず人間

Xアルゴリズムは会話最重視で、初動の人間のリアルタイム反応が効く。加えてXは「人間が画面をタップしていないと凍結されうる」と製品責任者が明言。AIは下書き・収集まで、投稿ボタンは人間が押す。

2

X APIを使わない

X APIは2026年2月に従量課金化(リンク付き投稿は$0.20/件)。収集はRSS/公開API(Hacker News・はてブ・Zenn・Qiita・Reddit・GitHub Trending)で足りる。認証不要・コスト0円。

3

下ごしらえを削るだけ

目的は「45分ルーチンを軽くする」こと。開発ファースト原則は維持。AIが素材を用意しても、書くのは実体験に基づく人間(AI丸出しの一般論は品質評価で不利なので捏造させない)。

毎日・週次の運用フロー

既存の45分ルーチンにそのまま乗せる。自動=AIが勝手にやる / 手動=人間がやる。

タイミング自動(AI)手動(人間)
毎朝7:00launchd → /collect がダイジェスト生成・ネタ帳更新—
朝 15〜20分起動時にSessionStartフックが今日のネタ+曜日テーマを表示Xで中堅アカウントに価値あるリプライ10件・リプ返し
昼 10分—リプ返し+ネタ帳に1行メモ
夜(開発後)10〜15分/draft today が下書き2〜3案1案を選んで手動投稿+直後15〜30分リプ返し
日曜バッチ 90分/review→/draft×5〜7→/weekly数値を貼る・下書き確認・予約投稿(X公式Web版)・スレッド投稿
✅ 最低ライン(これだけは死守)

リプ返5件 / 週3ポスト以上 / ネタ帳に1行。投稿後48時間は数字を見ない。SNS合計45分を超えたら閉じる。エネルギーの7割はリプライ(70/30)。

コマンド早見表

コマンドいつ使う何をする
/collect毎朝(自動)/ 手動でもRSS収集→界隈ダイジェスト→ネタ帳に型付きネタ追記
/draft today夜・開発後実体験から下書き2〜3案(NGチェック済・ドライな文体)
/draft 失敗談 <内容>型を指定したいとき指定の型・ネタで下書き
/weekly金 or 日週次振り返りスレッド(3〜6ポスト)の下書き
/review日曜15分投稿ログの数値からトップ3/来週の一手
trend-scout(agent)1トピック深掘りしたいとき調査して分析型ポストの素材(数字・出典・切り口)を返す

スキルは /名前 で呼ぶ。trend-scout はサブエージェントなので「〜を trend-scout で深掘りして」と依頼する。

セットアップ手順

ℹ️ 前提

Python3(macOS標準)と Claude Code CLI が使えること。追加インストールは不要(fetch_sources.py は標準ライブラリのみ)。作業ディレクトリは /Applications/smart_app_development/make_idea。

# 1) 収集の動作確認(手動・ネット取得できるか)
cd /Applications/smart_app_development/make_idea
python3 sns_marketing/scripts/fetch_sources.py --print

# 2) Claude Code内で /collect を手動実行 → ダイジェスト生成を確認
/collect

# 3) 無人実行(headless)の確認
bash sns_marketing/scripts/collect.sh
cat  sns_marketing/collect/cron.log      # 生成ログを確認

# 4) 毎朝7:00の自動実行を有効化(launchd)
cp   sns_marketing/scripts/com.hshinya.x-collect.plist ~/Library/LaunchAgents/
launchctl load  ~/Library/LaunchAgents/com.hshinya.x-collect.plist
launchctl start com.hshinya.x-collect    # 今すぐ1回テスト実行
⚠️ 権限について

無人実行では --dangerously-skip-permissions は使わない。collect.sh は /collect が使うツール(python3実行・ファイル読み書き)だけを --allowedTools と --permission-mode acceptEdits で許可している。モデルはコスト節約のため --model sonnet。

ℹ️ スケジュールの挙動

launchdはMacが起動中の7:00に実行する。スリープ/電源オフの時刻はスキップされ、次回起動後にまとめて走ることがある(仕様)。実行ログは collect/cron.log、launchd自身のエラーは collect/launchd.err.log。

停止・カスタマイズ

やりたいことどこを触る
収集ソース・キーワードを増減scripts/fetch_sources.py の SOURCES / KEYWORDS
実行時刻を変えるplist の Hour / Minute を編集して load し直す
自動実行を止めるlaunchctl unload ~/Library/LaunchAgents/com.hshinya.x-collect.plist
起動時ブリーフを消す.claude/settings.json の hooks を削除(このプロジェクトは他資料の作業もするため、うるさければオフに)
収集モデルを変えるcollect.sh の --model(sonnet→opus等)
下書きの文体を調整.claude/skills/draft/SKILL.md の「文体(固定ルール)」
🔭 将来: 界隈のXも自動で追いたくなったら

今はXアカウント追跡を入れていない(Nitter系は2026年時点で不安定・ブラウザ自動化はBANリスク)。界隈のXは人間が普段見て拾う運用。どうしても自動化するなら、リスクを理解した上でブラウザ自動化MCPを別途検証する(投稿は引き続き手動)。

前提と注意(2026年のX事情)

⚠️ 使う前に押さえる
  • X APIは2026年2月に従量課金化(新規はFreeティア廃止)。投稿$0.015/件・リンク付き$0.20/件・読み取り$0.005/件。→ 本システムはX APIを使わない。
  • 自動投稿はしない。X製品責任者が「人間が画面をタップしていないと(関連アカウントごと)凍結されうる」と明言。投稿・リプは人間が手動。
  • AIに一般論を書かせない。実体験の素材が無いときは /draft は人間に質問して止まる(捏造禁止)。AI丸出しポストは品質評価で不利。
  • 料金・API仕様・各ツールの無料枠は変動が激しい。利用直前に公式で再確認。

この自動化は発信プレイブックとロードマップの戦略を実行に落とすための道具。戦略の考え方はそちらを参照。
